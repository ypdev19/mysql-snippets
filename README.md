## 🤝 Contributing & Reporting Issues

Contributions are welcome!

**How to contribute:**

1. Fork the repo and create a new branch:

   ```bash
   git checkout -b snippet-name
   ```

2. Add your `.sql` file to the `/snippets` folder.
3. Commit your changes:

   ```bash
   git commit -m "Add snippet for XYZ"
   ```

4. Push your branch and open a Pull Request.

**Best practices for snippets:**

* Use descriptive, snake-case file names: `check-if-table-exists.sql`.
* Keep snippets focused on a single problem.
* Include examples of usage.
* If modifying existing snippets, explain changes in your PR.

**What to avoid:**

* Snippets without explanations or examples.
* Database-specific proprietary SQL.
* Full database dumps.

**Reporting bugs:**

* Open an [Issue](https://github.com/ypdev19/mysql-snippets/issues) with a clear description.

More contribution details: [CONTRIBUTING.md](./CONTRIBUTING.md)
